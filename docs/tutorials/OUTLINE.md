## 教程 01: OpenClaw 是什么？5 分钟理解 AI 网关的核心价值

**优先级**: P0  
**预估字数**: 3,500 字  
**预计完成时间**: 2 小时  
**目标读者**: 完全新手，想了解 OpenClaw 能做什么

### 大纲

#### 1.1 什么是 AI 网关？（500 字）
- 传统 AI 使用的痛点：多个平台、多个账号、多个界面
- 网关的概念：统一入口、集中管理
- OpenClaw 的定位：本地自托管 AI 网关

#### 1.2 OpenClaw 能解决什么问题？（800 字）
- 多渠道统一管理（WhatsApp/Telegram/Discord/微信等）
- 多 Agent 协同工作（工作/生活/学习分离）
- 数据本地化（隐私保护、无 API 限制）
- 自动化工作流（定时任务、消息路由）

#### 1.3 核心架构解析（1000 字）
- Gateway（网关）：消息中转站
- Agent（智能体）：实际处理逻辑
- Channel（渠道）：连接外部平台
- Node（节点）：扩展设备能力
- 架构图解（Mermaid 流程图）

#### 1.4 适用场景（700 字）
- 个人用户：统一 AI 助手
- 小团队：客服自动化
- 开发者：本地测试环境
- 企业：私有化部署

#### 1.5 与其他方案对比（500 字）
- vs 直接使用 ChatGPT/Claude
- vs 其他开源网关项目
- OpenClaw 的优势和劣势

# 附录

## A. 术语表

| 术语 | 解释 |
|------|------|
| Gateway | 网关，消息中转站 |
| Agent | 智能体，实际处理逻辑 |
| Channel | 渠道，连接外部平台 |
| Node | 节点，扩展设备能力 |
| Pairing | 配对，设备认证流程 |
| dmPolicy | 直接消息策略 |
| SecretRef | 密钥引用，安全存储 |

## B. 命令速查

### 安装和配置
```bash
openclaw onboard          # 运行向导
openclaw configure        # 配置向导
openclaw gateway status   # 检查网关状态
openclaw dashboard        # 打开控制面板
```

### 渠道管理
```bash
openclaw pairing list     # 查看配对请求
openclaw pairing approve  # 批准配对
openclaw channels login   # 登录渠道
```

### Agent 管理
```bash
openclaw agents add       # 添加 Agent
openclaw agents list      # 列出 Agent
openclaw agents switch    # 切换 Agent
```

### 节点管理
```bash
openclaw nodes status     # 节点状态
openclaw nodes describe   # 节点详情
openclaw nodes invoke     # 调用节点命令
```

## C. 配置文件示例

### 最小配置
```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace"
    }
  },
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      dmPolicy: "pairing"
    }
  }
}
```

### 完整配置示例
```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      model: {
        primary: "anthropic/claude-sonnet-4-5",
        fallbacks: ["openai/gpt-5.2"]
      }
    },
    list: [
      {
        id: "work",
        workspace: "~/.openclaw/workspace-work"
      },
      {
        id: "personal",
        workspace: "~/.openclaw/workspace-personal"
      }
    ]
  },
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      dmPolicy: "allowlist",
      allowFrom: ["tg:123456"]
    },
    whatsapp: {
      enabled: true,
      dmPolicy: "pairing"
    }
  },
  session: {
    dmScope: "per-channel-peer",
    reset: {
      mode: "daily",
      atHour: 4
    }
  },
  tools: {
    exec: {
      host: "local",
      security: "allowlist"
    },
    web: {
      search: {
        provider: "brave",
        apiKey: "YOUR_KEY"
      }
    }
  }
}
```
